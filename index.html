The escalation ladder

The organizing idea of this whole site. Before you reach for a large model, climb only as high as the problem forces you to. Tap a rung to see when to use it, whether it needs a GPU, and roughly what it costs.

Tap a rung above
Start at the bottom. Only climb when the rung below genuinely can’t solve the problem — each step up costs more compute, money, and latency.
The golden rule of an FDE: use the cheapest rung that actually works.
Why this framing
Interviewers (and customers) don’t reward reaching for the biggest model. They reward judgment: “Can a plain algorithm solve it? No → can classical ML? … do you need a GPU or is CPU fine?” This site teaches the concepts and the rung each one lives on.
