Multimodal AI

Everything so far has been mostly text. Multimodal models handle images, audio, and video too — and increasingly all at once. You don’t need research depth here, but you should be able to explain how it works and when to reach for it. This is a breadth overview to round out your map.

Where this sits on the ladder
Rung 3–4 for non-text data. Multimodal models are the deep-learning/LLM rungs applied to images and audio. The same discipline holds: for a narrow vision task a small specialized model on a GPU may beat a giant multimodal LLM on cost and latency — climb only as high as the task needs.

CLIP & the shared embedding space

The foundational idea. CLIP trains an image encoder and a text encoder together so that an image and its caption land at the same point in a shared embedding space. Once images and text live in one space, you can compare them directly — “which of these images best matches this sentence?” becomes a nearest-neighbor lookup. It’s the multimodal version of the embeddings you already know.

CLIP pulls matching image/text pairs together and pushes mismatches apart in one shared space — so cross-modal similarity is just a dot product.

Vision-language models (VLMs)

Modern multimodal LLMs (GPT-4o, Claude, Gemini, LLaVA) accept images and text in the same prompt. Mechanically, an image is encoded into embeddings (“visual tokens”) that are fed into the LLM alongside text tokens — so the transformer attends over both. That’s what powers Visual Question Answering (VQA) — “what’s wrong in this diagram?”, reading a screenshot, extracting a table from a photo. For an FDE this unlocks document/image understanding without a separate vision pipeline.

Diffusion (text → image)

Image generation (Stable Diffusion, DALL·E, Midjourney) works differently from LLMs. Diffusion models learn to reverse a noising process: start from pure noise and iteratively denoise it, step by step, guided by your text prompt, until a coherent image emerges. So generation is not autoregressive token-by-token — it’s progressive refinement of the whole image. That’s the one-line contrast interviewers look for: LLMs predict the next token; diffusion models denoise toward an image.

Speech: STT & TTS

  • Speech-to-text (STT / ASR) — transcribe audio to text (e.g. Whisper). Often the front door of a voice pipeline: transcribe → process with an LLM → respond.
  • Text-to-speech (TTS) — synthesize natural speech from text; the back door of the same pipeline.
  • Increasingly, models are natively speech-to-speech, skipping the text round-trip for lower latency in voice assistants.

Multimodal RAG

RAG isn’t limited to text. With a shared embedding space (CLIP-style) you can retrieve images by a text query, or answer questions over documents that mix text, tables, charts, and photos. Two common patterns: embed everything into one multimodal space and retrieve across modalities; or caption/describe images with a VLM and retrieve over that text. Same RAG mechanics (see RAG) — just more modalities in the index.

Interview Q&A

coreWhat is CLIP and why is it useful?

CLIP jointly trains an image encoder and a text encoder with a contrastive objective so that matching image–caption pairs are close in a shared embedding space and mismatches are far apart. Because images and text end up in one space, you can measure cross-modal similarity with a dot product — enabling zero-shot image classification, text-to-image search, and multimodal retrieval. It’s the multimodal extension of ordinary embeddings.

coreHow is a diffusion model different from an LLM?

An LLM is autoregressive — it generates text one token at a time, each conditioned on the previous ones. A diffusion model generates an image by starting from random noise and iteratively denoising it over many steps, guided by the text prompt, refining the whole image at once rather than emitting pieces sequentially. Different objective (denoising vs next-token) and different output structure (whole image vs a token stream).

seniorHow do multimodal LLMs actually “see” an image?

An image is passed through a vision encoder that turns it into a sequence of embedding vectors — effectively “visual tokens” — which are projected into the LLM’s embedding space and fed in alongside the text tokens. The transformer then attends over both modalities together, so it can reason about the image in the context of the question. The LLM isn’t processing pixels directly; it’s consuming image embeddings the way it consumes word embeddings.

trapYou need to classify 1M product images into 20 categories. Multimodal LLM?

Probably not the frontier multimodal LLM — that’s expensive and slow at that volume. For a fixed set of categories with labeled data, a specialized image classifier (a fine-tuned vision model) on a GPU is cheaper, faster, and likely more accurate. CLIP enables a strong zero-shot baseline if you lack labels. Reserve a VLM for open-ended visual understanding, not a high-volume, fixed-label classification task — the ladder applies to vision too.

seniorHow would you build search over documents that contain text and charts?

Two viable patterns. One: use a multimodal embedding model so text and images share a space, and retrieve across both directly. Two (often simpler and robust): use a VLM to caption/describe each image or chart, index that text alongside the document text, and run ordinary text RAG. Then a multimodal LLM answers over the retrieved mix. I’d start with the caption-and-index approach for reliability, and evaluate faithfulness and retrieval quality just like text RAG.

Code: CLIP image/text similarity

Zero-shot classification with CLIP: encode an image and several candidate captions into the shared space, and pick the closest. No training — the shared embedding space does the work.

pythonzero-shot image classification with CLIP
# CLIP: match an image against text prompts in a shared embedding space.
# pip install transformers torch pillow
from transformers import CLIPProcessor, CLIPModel
from PIL import Image
import torch

model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
proc  = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

image = Image.open("photo.jpg")
# Candidate labels as natural-language prompts (this is the "zero-shot" part).
labels = ["a photo of a cat", "a photo of a dog", "a photo of a truck"]

# Encode BOTH modalities into the same space and let CLIP score the matches.
inputs = proc(text=labels, images=image, return_tensors="pt", padding=True)
with torch.no_grad():
    out = model(**inputs)
    # logits_per_image: similarity of the image to each text prompt.
    probs = out.logits_per_image.softmax(dim=1)[0]

for label, p in sorted(zip(labels, probs.tolist()), key=lambda x: -x[1]):
    print(f"{p:.2%}  {label}")
# The highest-probability caption is CLIP's prediction — no labeled training data,
# just similarity in the shared image/text space. Same idea powers text->image search.
The takeaway
Multimodal is the same toolbox — embeddings, transformers, RAG — extended to pixels and audio. If you can explain CLIP’s shared space and “LLMs predict tokens, diffusion denoises images,” you can hold your own on multimodal without going down a research rabbit hole.
