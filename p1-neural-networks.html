What deep learning is

What it is

Deep learning is machine learning with neural networks that have many layers, learning their own features from raw data. “Deep” just means “many layers stacked” — and depth is what lets the network build up from edges to shapes to objects, or from letters to words to meaning, with no human feature engineering.

This is the key contrast with the previous page. Classical machine learning needs you to hand it good features; a deep network learns the features itself from the raw pixels, waveform, or tokens. That’s a superpower for unstructured data where the right features are impossible to state by hand — and usually wasted effort on a tidy spreadsheet, where a gradient-boosted tree already wins. The price of that superpower is data and compute: deep learning wants lots of labelled examples and a GPU to train.

Bottom line
Deep learning trades interpretability and cheap CPU training for the ability to learn features from raw, unstructured data. Climb here when that trade actually pays — not by default.

From neuron to network

What it is

An artificial neuron is a tiny function: weighted sum → add a bias → apply a non-linear activation, written a = σ(w·x + b). Stack neurons into layers and layers into a network — a Multi-Layer Perceptron (MLP) — and you get a function that can approximate almost anything.

Each neuron multiplies its inputs by learned weights, adds a bias, and passes the result through an activation σ that bends the line into a curve. One neuron is a glorified logistic regression; the magic is in composition. Wire many together across layers and the network gains the universal approximation property — enough capacity to represent essentially any continuous function. The catch: that’s an existence guarantee. It says nothing about how much data or compute you need to actually find those weights, which is where training comes in.

Signal flows left → right (the forward pass). Every edge is a learned weight; every node applies an activation.
Bottom line
A network is just neurons composed across layers. Its power comes from depth and non-linearity; its usefulness comes entirely from training the weights — which the rest of this page explains.

The forward pass

What it is

The forward pass is running an input through the network to get a prediction — layer by layer, each one a matrix multiply followed by an activation.

For one layer: h = σ(W·x + b), then that h becomes the input to the next layer, and so on to the output. That’s all inference is. Because it’s dominated by big matrix multiplications, a GPU — essentially a massive parallel multiply-add machine — runs it orders of magnitude faster than a CPU. That single fact is why deep learning lives on the GPU line of the ladder, and it’s the same operation that later powers transformer inference.

Bottom line
The forward pass is “just matrix math.” Cheap per-layer, embarrassingly parallel, GPU-friendly — and the exact computation you’ll optimise later for large-model inference.

Backpropagation & gradient descent

What it is

Backpropagation is the chain rule of calculus run backwards through the network to compute how much each weight contributed to the error. Gradient descent is the follow-up step that nudges every weight a little in the direction that reduces the error. Together they are how a network learns.

Training answers one question, repeatedly: how should I nudge each weight to reduce the loss?

  1. Forward pass — compute the prediction and the loss (how wrong it is).
  2. Backward pass (backpropagation) — apply the chain rule from the loss back through every layer to get each weight’s gradient (its share of the blame), reusing intermediate results so it’s efficient.
  3. Update — step each weight a little downhill: w ← w − η·∇w, where η (eta) is the learning rate.

Repeat over many mini-batches and many passes through the data (epochs). Gradient descent is literally “roll downhill on the loss surface,” and the learning rate sets your step size.

The ball (a weight) rolls down the loss surface toward the minimum. The learning rate sets its step size: too big and it overshoots; too small and it crawls.
Bottom line
Backprop assigns blame; gradient descent acts on it. Forward pass → loss → backward pass → update, looped over the data, is the entire training algorithm behind every deep model — including transformers.

Activations, losses & optimizers

What it is

These are the three choices that shape training: the activation (the non-linearity in each neuron), the loss (what “wrong” means), and the optimizer (how gradients get turned into weight updates).

  • Activation — the non-linearity that lets networks model curves, not just lines. ReLU (Rectified Linear Unit), max(0, x), is the default hidden-layer choice; GELU (Gaussian Error Linear Unit) / SiLU (Sigmoid Linear Unit) in transformers; softmax on the output for multi-class probabilities.
  • Loss — what “wrong” means. Cross-entropy for classification, MSE (Mean Squared Error) for regression. The loss is the thing backpropagation differentiates.
  • Optimizer — how you apply gradients. Plain SGD (Stochastic Gradient Descent) works; Adam / AdamW adapt the step size per-parameter and converge faster — the modern default, especially for transformers.
Vanishing / exploding gradients
Deep stacks can lose or blow up the gradient signal on the way back. The fixes you should name: ReLU-family activations, residual connections, batch / layer normalization, and careful weight initialization — several of which reappear inside the transformer block.
Bottom line
Activation adds the curve, loss defines the target, optimizer drives the descent. Sensible defaults: ReLU (or GELU in transformers), cross-entropy or MSE, and AdamW.

Interview Q&A

coreExplain backpropagation in one breath.

It’s the chain rule applied backwards through the network. After a forward pass computes the loss, backprop propagates the derivative of the loss with respect to each weight from the output layer back to the input, reusing intermediate results. Those gradients tell the optimizer which direction and how much to nudge every weight.

trapWhy do we need a non-linear activation at all?

Without one, stacking layers is pointless: a composition of linear maps is still a single linear map, so a 50-layer network would have the exact same expressive power as one linear layer. The non-linearity (ReLU, GELU…) is what lets the network model curved decision boundaries and complex functions.

seniorSGD vs Adam — when does the choice matter?

Adam adapts a per-parameter learning rate from running estimates of the gradient’s mean and variance, so it converges fast with little tuning — the safe default, especially for transformers (as AdamW). Well-tuned SGD with momentum can generalize slightly better on some vision tasks and uses less memory. In practice: reach for AdamW first; consider SGD if you’re squeezing out the last bit of generalization with a tuning budget.

coreWhen would you not use deep learning?

For tabular/structured data with modest volume, when I need interpretability, or when a CPU budget matters — which covers a huge share of real business problems. Gradient-boosted trees usually match or beat a neural net there, train in seconds, and are explainable. Deep learning earns its cost on unstructured data (images, audio, language) or when classical models plateau.

seniorWhat is the universal approximation theorem, and why doesn’t it mean “always use a neural net”?

It says a sufficiently wide network can approximate any continuous function to arbitrary precision. But it’s an existence result — it says nothing about how much data, compute, or tuning you need to actually find those weights, or whether the result generalizes. On tabular data a boosted tree often reaches the same accuracy with a fraction of the cost, so “can represent it” ≠ “is the right tool.”

trapWhat are vanishing gradients, and how do modern nets avoid them?

In a deep stack, repeatedly multiplying small derivatives during backprop can shrink the gradient toward zero before it reaches the early layers, so they stop learning (or explode if the factors are large). The standard fixes are ReLU-family activations, residual/skip connections, batch or layer normalization, and careful initialization — the same ingredients that let transformers stack dozens of layers.

Code: a neural net with a hand-written training loop

The same tabular task as the classical baseline, but as a neural network — with the full training loop spelled out so you can see the forward pass, loss, backpropagation, and optimizer step. This loop is the exact pattern behind every deep model, including transformers.

pythonrung 3 — an MLP with a hand-written training loop (PyTorch)
# Rung 3: a neural net with the training loop spelled out, so you SEE every step.
# For plain tabular data this is usually overkill — but the loop is the exact
# pattern behind every deep model, including transformers.
import torch
import torch.nn as nn
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split

# Same style of tabular data as the classical baseline, converted to tensors.
X, y = make_classification(n_samples=5000, n_features=20, n_informative=8, random_state=0)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, random_state=0)
Xtr = torch.tensor(X_tr, dtype=torch.float32)
ytr = torch.tensor(y_tr, dtype=torch.float32).unsqueeze(1)   # shape (N, 1)
Xte = torch.tensor(X_te, dtype=torch.float32)

# 1) Define the network: input(20) -> hidden(64) -> ReLU -> output(1).
model = nn.Sequential(
    nn.Linear(20, 64),   # weighted sum + bias  (W·x + b)
    nn.ReLU(),           # non-linearity — without this, layers collapse to one linear map
    nn.Linear(64, 1),    # single logit output
)

# 2) Loss + optimizer. BCEWithLogits = sigmoid + binary cross-entropy in one stable op.
loss_fn   = nn.BCEWithLogitsLoss()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3)   # adaptive, the modern default

# 3) The training loop — forward, loss, backward (backprop), step. Repeat.
for epoch in range(200):
    optimizer.zero_grad()          # clear gradients from the previous step
    logits = model(Xtr)            # FORWARD pass: predictions
    loss   = loss_fn(logits, ytr)  # how wrong are we?
    loss.backward()                # BACKPROP: chain-rule gradients for every weight
    optimizer.step()               # UPDATE: w <- w - lr * grad
    if epoch % 50 == 0:
        print(f"epoch {epoch:3d}  loss {loss.item():.4f}")

# 4) Evaluate (no gradients needed at inference).
with torch.no_grad():
    prob = torch.sigmoid(model(Xte)).squeeze()     # logits -> probabilities
    acc  = ((prob > 0.5).int() == torch.tensor(y_te)).float().mean()
print(f"MLP accuracy: {acc:.3f}")
# On tabular data this is usually within a hair of the gradient-boosted tree —
# which is exactly why the cheap CPU model usually wins the deployment decision.
