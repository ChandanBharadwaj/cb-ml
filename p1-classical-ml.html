Classical Machine Learning

“Modern” does not mean “always a neural network.” On a spreadsheet — rows of customers, transactions, sensor readings — a handful of classical models remain the strongest, cheapest, most explainable option. This page defines each one from scratch and gives you the judgment to reach for it before you ever spend on a Graphics Processing Unit (GPU).

What classical machine learning is

What it is

Classical machine learning is the family of models that learn from features you provide, rather than learning the features themselves. They’re the pre-deep-learning workhorses — regression, trees, nearest-neighbours, clustering — and they still dominate structured, tabular problems.

The defining trait: you do the feature engineering. You decide that “days since last purchase” or “transactions per week” are the useful signals, and the model finds the relationship between those features and the label. (Deep learning, on the next page, flips this — it learns the features itself, which is powerful for images and text but usually overkill for a spreadsheet.) In exchange you get models that are fast, cheap, interpretable, and hard to beat on tabular data.

The line to remember in an interview
“For tabular data I start with gradient-boosted trees. They beat deep learning on most structured datasets, train on a CPU in seconds, and give me feature importances I can show a stakeholder. I only move to deep learning when the signal is in unstructured data or the trees plateau.”

Linear & logistic regression

What it is

Linear regression predicts a number by fitting a straight-line relationship — a weighted sum of the features. Logistic regression adapts that same weighted sum to predict a probability for a yes/no class, by squashing the output through an S-shaped curve.

These are the first thing you should try, and the sanity check every other model must beat. Linear regression models y = w·x + b: each feature gets a weight, and the prediction is their sum. Logistic regression wraps that sum in a sigmoid so the output lands between 0 and 1 and reads as a probability — “an 82% chance this customer churns.” Both train in milliseconds and hand you interpretable coefficients: you can point at a feature and say exactly how much it moves the prediction, which regulators and stakeholders love.

Bottom line
Reach for regression when you need a fast, explainable baseline, or when a simple linear relationship is genuinely enough. If it already clears the bar, you may be done — that’s a win, not a compromise.

Decision trees

What it is

A decision tree is a flowchart of yes/no questions learned from data. Each internal node splits on a feature (“income > 50k?”), and you follow the branches down to a leaf that holds the prediction.

Trees are the building block behind the two strongest tabular models (forests and boosting), so it’s worth understanding them on their own. Training greedily picks, at each node, the feature and threshold that best separate the data. Trees are wonderfully intuitive and need no feature scaling, and they naturally capture non-linear rules and interactions between columns. Their weakness: a single deep tree overfits — memorising the training set. The fix that made trees dominant is to combine many of them, which is exactly what the next two sections do.

Bottom line
One tree is interpretable but fragile. Its real value is as the unit that random forests and gradient boosting assemble into something powerful.

Random forests

What it is

A random forest is an ensemble of many decision trees, each trained on a random slice of the rows and columns, whose votes are averaged. The randomness makes the trees disagree in different ways, and averaging cancels out their individual mistakes.

This is bagging (bootstrap aggregating): grow hundreds of trees in parallel, each on a bootstrapped sample, then average. Because errors are largely independent, the forest is far more stable than any single tree and rarely overfits badly. It needs little tuning, handles mixed feature types, and gives you feature importances for free — a robust, low-effort baseline.

Bottom line
Random forest is the “set it and forget it” tabular model: strong out of the box, hard to misuse. When you want the last few points of accuracy, boosting usually edges it out.

Gradient-boosted trees

What it is

Gradient-boosted trees (a Gradient-Boosted Machine, or GBM) build an ensemble of shallow trees one at a time, where each new tree is trained to correct the leftover errors — the residuals — of all the trees so far.

Where a random forest builds trees independently and averages, boosting builds them sequentially and adds them up. It’s gradient descent in “function space”: each step adds the tree that most reduces the remaining error. That additive correction is what captures the messy non-linear interactions between columns that tabular data is full of — without you hand-crafting them — and it’s why libraries like XGBoost (eXtreme Gradient Boosting) and LightGBM (Light Gradient-Boosting Machine) win a large share of Kaggle competitions on structured data.

Boosting = additive correction. The red error bar shrinks as each shallow tree cleans up the previous ones’ mistakes.
The trade-off
Because boosting fits errors sequentially, it can overfit if you add too many trees or set the learning rate too high. Random forests are more forgiving; boosting rewards a little tuning (learning rate, tree count, depth) with the best tabular accuracy you’ll get.
Bottom line
Gradient-boosted trees are the tabular champion and the baseline every deep model must beat. Start here; only climb the ladder if this plateaus.

k-nearest-neighbours & k-means

What it is

k-Nearest Neighbours (k-NN) classifies a new point by the majority vote of its k closest examples — no training at all, just “who do you resemble?” k-means is its unsupervised cousin: it groups unlabelled data into k clusters around learned centre points.

Both rest on the idea of distance in feature space. k-NN is supervised and lazy — it stores the data and, at prediction time, finds the nearest neighbours and copies their label; great for quick similarity logic, but slow at scale. k-means is unsupervised — it discovers natural groupings (customer segments, say) by iteratively moving cluster centres to the middle of the points assigned to them. The same “nearest vector” intuition reappears later, at scale, as vector search over embeddings.

Bottom line
Reach for k-NN when “most similar past examples” is a good answer and the dataset is small; reach for k-means when you want to discover groups in unlabelled data.

Interview Q&A

coreRandom forest vs gradient boosting — what’s the core difference?

A random forest builds many deep trees independently and in parallel on random subsets, then averages them (bagging) — it reduces variance and is hard to misuse. Gradient boosting builds shallow trees sequentially, each correcting the previous ensemble’s residual errors — it reduces bias and usually gives higher accuracy, at the cost of more careful tuning to avoid overfitting.

coreWhy do gradient-boosted trees dominate tabular data?

They capture complex non-linear interactions between columns without manual feature crossing, handle mixed types and missing values, train fast on a CPU, and expose feature importances. For structured data they typically match or beat neural networks with a fraction of the cost and effort.

trapWhy does a single decision tree overfit, and how do ensembles fix it?

A deep tree keeps splitting until it isolates individual training rows — memorising noise. An ensemble fixes it two ways: bagging (random forests) averages many independent trees so their errors cancel; boosting uses shallow, deliberately weak trees and adds them gradually, so no single tree can memorise. Both trade a little interpretability for a lot of generalisation.

seniorWhen would you still pick logistic regression over a boosted model?

When interpretability is the requirement (regulated lending, clinical settings) and you need to state exactly how each feature affects the outcome; when the dataset is tiny and a complex model would overfit; or when you need a transparent, well-calibrated baseline to benchmark everything else against. Simpler is a feature, not a limitation.

coreIs k-means supervised or unsupervised, and what does “k” mean?

Unsupervised — it works on unlabelled data. “k” is the number of clusters you ask it to find; it’s a hyperparameter you choose (often guided by the “elbow” in the within-cluster error). k-NN, confusingly named, is the supervised one, where k is the number of nearest neighbours that vote on a label.

Code: the tabular baseline you run first

The honest way to decide whether you even need deep learning is to run the classical baseline first and note the number. This is the model you should almost always try before touching a GPU:

pythonrung 2 — gradient-boosted trees (scikit-learn)
# Rung 2: the tabular baseline you run FIRST. Trains on a CPU in seconds.
# (HistGradientBoosting is sklearn's built-in LightGBM-style booster; for the
#  real thing use xgboost.XGBClassifier or lightgbm.LGBMClassifier — same idea.)
from sklearn.datasets import make_classification
from sklearn.ensemble import HistGradientBoostingClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, roc_auc_score

# A stand-in tabular dataset: 20 features, some informative, some noise.
X, y = make_classification(n_samples=5000, n_features=20, n_informative=8, random_state=0)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, random_state=0)

# One line to a strong model — no scaling, no GPU, handles feature interactions.
clf = HistGradientBoostingClassifier(max_iter=200, learning_rate=0.08, max_depth=None)
clf.fit(X_tr, y_tr)                                   # trains in ~1 second on a CPU

pred  = clf.predict(X_te)
proba = clf.predict_proba(X_te)[:, 1]                 # P(class = 1), for AUC/threshold tuning
print(f"accuracy: {accuracy_score(y_te, pred):.3f}")
print(f"ROC-AUC : {roc_auc_score(y_te, proba):.3f}")  # threshold-independent quality
# This number is your bar. Only climb to deep learning if you must beat it.
